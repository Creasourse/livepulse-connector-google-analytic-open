## 依赖中间件安装

### 1. Nginx 安装与配置

```bash
# 安装 Nginx
apt install nginx -y

# 将 www-data 加入 ga4_conn 用户组（用于处理文件权限）
sudo usermod -a -G ga4_conn www-data
```

### 2. PostgreSQL 15 安装与配置

#### 2.1 添加官方仓库并安装

```bash
# 创建仓库配置文件
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# 导入签名密钥
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# 更新软件包列表
sudo apt update

# 安装 PostgreSQL 15 服务端和客户端
sudo apt install postgresql-15 postgresql-client-15 -y
```

#### 2.2 服务管理

```bash
# 检查服务状态
sudo systemctl status postgresql@15-main

# 设置开机自启
sudo systemctl enable postgresql@15-main

# 启动 / 停止服务
sudo systemctl start postgresql@15-main
sudo systemctl stop postgresql@15-main
```

#### 2.3 数据库初始化

```bash
# 切换至 postgres 用户并进入 psql 终端
sudo -u postgres psql
```

在 psql 终端中执行以下 SQL：

```sql
-- 创建用户 ga4_conn 并设置密码
CREATE USER ga4_conn WITH PASSWORD 'abc123';

-- 创建数据库并指定所有者
CREATE DATABASE ga4_connector_db OWNER ga4_conn;

-- 授权
GRANT ALL PRIVILEGES ON DATABASE ga4_connector_db TO ga4_conn;
```

#### 2.4 配置远程访问

编辑 `pg_hba.conf` 允许远程连接：

```bash
vi /etc/postgresql/15/main/pg_hba.conf
```

在文件末尾追加（或修改）以下行：

```
host    all             all             0.0.0.0/0               md5
```

编辑 `postgresql.conf` 监听所有地址：

```bash
vi /etc/postgresql/15/main/postgresql.conf
```

修改配置：

```
listen_addresses = '*'
```

配置完成，请重启 PostgreSQL 服务。

#### 2.5 执行初始化数据库脚本

```bash
# 执行建表脚本
psql -U ga4_conn -d ga4_connector_db -f src/main/resources/db/schema.sql
```

### 3. Nacos 安装与配置

#### 3.1 下载与启动

从官网下载 `nacos-server-2.5.1.tar.gz`，解压并以单机模式启动：

```bash
# 解压
tar -zxvf nacos-server-2.5.1.tar.gz

# 进入 bin 目录
cd nacos/bin

# 单机模式启动
./startup.sh -m standalone
```

#### 3.2 开启鉴权

编辑 `conf/application.properties`，在末尾追加以下配置以开启认证：

```properties
nacos.core.auth.system.type=nacos
nacos.core.auth.enabled=true
nacos.core.auth.console.enabled=true
nacos.core.auth.caching.enabled=true
nacos.core.auth.enable.userAgentAuthWhite=false
nacos.core.auth.server.identity.key=ZGhzc2tueGhqZHhtanVlbmpkZGpkamRqZGo=
nacos.core.auth.server.identity.value=ZGhzc2tueGhqZHhtanVlbmpkZGpkamRqZGo=
nacos.core.auth.plugin.nacos.token.secret.key=ZGhzc2tueGhqZHhtanVlbmpkZGpkamRqZGpkZGRkZGQ=
```

#### 3.3 修改管理员密码

请替换 IP 和端口后执行：

```bash
curl -X POST 'http://192.168.1.10:8848/nacos/v1/auth/users/admin' -d 'password=123456'
```

#### 3.4 执行 Nacos 初始化配置文件

登录 Nacos 控制台: `http://nacos-server:8848/nacos`
默认账号密码: `nacos/nacos`（修改后使用新密码）

在命名空间中创建配置文件（参见 [Nacos 配置](#nacos-配置)）。

### 4. Redis 安装与配置

#### 4.1 安装

```bash
apt install redis-server -y
```

#### 4.2 配置密码

编辑配置文件 `/etc/redis/redis.conf`，在末尾追加密码设置：

## Docker 及 Docker Compose 安装

### 1. 安装 Docker Engine

#### 1.1 卸载旧版本

```bash
sudo apt remove docker docker-engine docker.io containerd runc -y
```

#### 1.2 安装依赖工具

```bash
sudo apt update
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
```

#### 1.3 添加 Docker 官方 GPG 密钥

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```

#### 1.4 添加 Docker 软件源

```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

#### 1.5 安装 Docker

```bash
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y
```

### 2. 安装 Docker Compose (插件版)

#### 2.1 创建目录

```bash
mkdir -p ~/.docker/cli-plugins
```

#### 2.2 下载二进制文件

```bash
curl -SL https://github.com/docker/compose/releases/download/v2.39.3/docker-compose-linux-x86_64 -o ~/.docker/cli-plugins/docker-compose
```

#### 2.3 赋予执行权限

```bash
chmod +x ~/.docker/cli-plugins/docker-compose
```

#### 2.4 验证安装

```bash
docker compose version
```

### 3. 可选配置（推荐）

#### 3.1 免 sudo 执行 Docker

```bash
sudo usermod -aG docker $USER
```

注意：执行此命令后需**注销并重新登录**（或重启服务器），配置才会生效。

#### 3.2 验证运行状态

```bash
sudo systemctl status docker
```

## 应用部署

### 1. 准备配置文件

创建 `.env` 文件（与 docker-compose.yml 同目录）：

```bash
# Java 应用参数
JAVA_OPTS=-Xms512m -Xmx1024m

# Spring 配置
SPRING_PROFILES_ACTIVE=prod

# Nacos 配置
SPRING_CLOUD_NACOS_SERVER_ADDR=192.168.1.10:8848
SPRING_CLOUD_NACOS_CONFIG_GROUP=cs
SPRING_CLOUD_NACOS_DISCOVERY_GROUP=cs
SPRING_CLOUD_NACOS_CONFIG_NAMESPACE=cs
SPRING_CLOUD_NACOS_DISCOVERY_NAMESPACE=cs
SPRING_CLOUD_NACOS_USERNAME=nacos
SPRING_CLOUD_NACOS_PASSWORD=nacos

# GA4 配置
GA4_PROJECT_ID=your-gcp-project-id
GA4_DATASET_ID=analytics_123456789
GA4_USE_DEFAULT_CREDENTIALS=false

# 端口配置
APP_PORT_GA4=23006

# 镜像标签
TAG=latest
```

### 2. Docker Compose 配置说明

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  ga4-connector:
    image: livepulse/ga4-connector:${TAG}
    container_name: ga4-connector
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "${APP_PORT_GA4}:23006"
    env_file:
      - .env
    volumes:
      - ./volumes/app/log:/data/app/logs
      - /path/to/key.json:/app/config/key.json:ro
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:23006/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
```

### 3. 目录结构

```
livepulse-connector-google-analytic-open/
├── docker-compose.yml              # Docker Compose 配置文件
├── .env                            # 环境变量配置
├── Dockerfile                      # 镜像构建文件
├── ga4_conn_nginx.conf             # Nginx 配置文件
└── volumes/
    └── app/
        └── log/                    # 应用日志挂载目录
```

### 4. 构建并启动服务

## Google Cloud 配置

### GA4 BigQuery 导出配置

确保您的 GA4 属性已配置 BigQuery 数据导出：

1. 在 Google Analytics 中，进入 Admin > Property > Data Streams
2. 选择您的数据流
3. 点击 BigQuery Linking
4. 创建新的 BigQuery 链接
5. 选择项目和数据集
6. 保存配置

### Google Cloud 认证配置

有两种认证方式：

#### 方式一：服务账号密钥（开发环境）

1. 创建服务账号：
   ```bash
   gcloud iam service-accounts create ga4-connector \
     --display-name="GA4 Connector Service Account"
   ```

2. 授予 BigQuery 访问权限：
   ```bash
   gcloud projects add-iam-policy-binding PROJECT_ID \
     --member="serviceAccount:ga4-connector@PROJECT_ID.iam.gserviceaccount.com" \
     --role="roles/bigquery.dataViewer"
   ```

3. 创建并下载密钥：
   ```bash
   gcloud iam service-accounts keys create key.json \
     --iam-account=ga4-connector@PROJECT_ID.iam.gserviceaccount.com"
   ```

4. 将密钥文件挂载到容器中（已在 docker-compose.yml 中配置）

#### 方式二：Workload Identity（生产环境推荐）

1. 配置 Workload Identity
2. 在 Kubernetes 中配置 ServiceAccount 和 GCP IAM 绑定
3. 设置环境变量: `GA4_USE_DEFAULT_CREDENTIALS=true`

## Nginx 配置

### 1. 配置文件说明

配置文件：`ga4_conn_nginx.conf`

**域名**：`ga4-connector.livepulse.com.cn`

**端口**：80（HTTP） / 443（HTTPS，可选）

**关键配置**：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| client_max_body_size | 300m | 最大请求主体大小 |
| proxy_connect_timeout | 600s | 连接超时 |
| proxy_read_timeout | 600s | 读取超时 |
| proxy_send_timeout | 600s | 发送超时 |

### 2. 路由配置

#### 2.1 GA4 Connector API

```
location /ga4-api {
    rewrite ^/ga4-api/(.*) /$1 break;
    proxy_pass http://localhost:23006;
    ...
}
```

请求示例：
- `http://ga4-connector.livepulse.com.cn/ga4-api/xxx` → `http://localhost:23006/xxx`

### 3. 部署 Nginx 配置

```bash
# 复制配置文件到 Nginx 配置目录
cp ga4_conn_nginx.conf /etc/nginx/sites-available/ga4-connector

# 创建软链接启用站点
ln -s /etc/nginx/sites-available/ga4-connector /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
nginx -s reload
```

## 故障排查

### 1. 容器无法启动

```bash
# 查看容器日志
docker compose logs ga4-connector

# 检查镜像构建
docker compose build --no-cache

# 检查端口占用
netstat -tulpn | grep 23006
```

### 2. 服务无法访问

```bash
# 检查容器运行状态
docker ps

# 检查服务端口
docker compose ps

# 检查网络连接
docker network ls
docker network inspect livepulse-connector-google-analytic-open_app-network
```

### 3. 数据库连接失败

```bash
# 检查 PostgreSQL 状态
systemctl status postgresql@15-main

# 测试数据库连接
psql -h localhost -U ga4_conn -d ga4_connector_db

# 检查防火墙规则
iptables -L -n | grep 5432
```

### 4. BigQuery 连接失败

**检查项**：
- 服务账号密钥文件路径是否正确
- 密钥文件权限是否正确（建议 600）
- 项目 ID 和数据集 ID 是否正确
- 网络是否可以访问 Google Cloud API

```bash
# 测试 BigQuery 连接
curl http://localhost:23006/actuator/health
```

### 5. Nacos 连接失败

```bash
# 检查 Nacos 状态
curl http://localhost:8848/nacos/v1/console/health/readiness

# 查看服务注册情况
curl http://localhost:8848/nacos/v1/ns/instance/list?serviceName=google-analytic-open-connector-server
```

### 6. 同步任务未执行

**检查项**：
- 调度任务是否启用（ga4.scheduled.enabled=true）
- Cron 表达式是否正确
- GA4 属性是否已启用（enabled=true）
- 日志中是否有错误信息
